# CallCodeX 公网访问方案

## 为什么 `172.23.192.1` 手机访问不了

`172.23.192.1` 是 Windows 的 `vEthernet` 虚拟网卡地址。它不是公网地址，也通常不是你家里 Wi-Fi 局域网里的地址。

如果 iPhone 和 Windows 在同一个 Wi-Fi 下，可以使用 Windows 当前的局域网地址，通常类似：

```text
192.168.x.x
```

访问形式：

```text
http://192.168.x.x:8787/?token=你的口令
```

如果人在外面，这个 `192.168.x.x` 地址也不能直接访问。

## 推荐方案一：Tailscale，非公网暴露

这是最推荐的日常使用方案。

特点：

- 不开放路由器端口。
- 不需要公网 IP。
- iPhone 和 Windows 都安装 Tailscale。
- 手机通过 Windows 的 `100.x.x.x` Tailscale 地址访问。

访问形式：

```text
http://100.x.x.x:8787/?token=你的口令
```

这不是公网访问，但人在外面也能用。

## 推荐方案二：Cloudflare Tunnel + Cloudflare Access，公网域名访问

如果你明确想用公网 HTTPS 地址访问，推荐 Cloudflare Tunnel。

特点：

- 不需要路由器端口映射。
- 不需要家里有公网 IP。
- Windows 主动连到 Cloudflare，外部用户访问 Cloudflare 域名。
- 可以用 Cloudflare Access 加邮箱验证码或账号登录保护。

访问形式：

```text
https://codex.example.com
```

安全建议：

- 不要直接裸奔公开 CallCodeX。
- 必须保留 CallCodeX 的 `AUTH_TOKEN`。
- 最好再加 Cloudflare Access，只允许你的邮箱访问。

基本命令示例：

```powershell
winget install --id Cloudflare.cloudflared
cloudflared tunnel login
cloudflared tunnel create callcodex
cloudflared tunnel route dns callcodex codex.example.com
```

创建 Cloudflare Tunnel 配置文件时，把公网域名转发到本地服务：

```yaml
tunnel: callcodex
credentials-file: C:\Users\你的用户名\.cloudflared\隧道ID.json

ingress:
  - hostname: codex.example.com
    service: http://127.0.0.1:8787
  - service: http_status:404
```

启动：

```powershell
cloudflared tunnel run callcodex
```

临时测试可以用 Quick Tunnel：

```powershell
cloudflared tunnel --url http://127.0.0.1:8787
```

它会生成一个临时 `trycloudflare.com` 地址。临时地址适合测试，不适合长期使用。

## 可选方案三：Tailscale Funnel，公网地址访问

Tailscale Funnel 可以把本机服务发布到公网 URL。

特点：

- 不需要路由器端口映射。
- 不要求访问者安装 Tailscale。
- 需要在 Tailscale 管理后台启用 Funnel。

大致命令：

```powershell
tailscale serve --bg 8787
tailscale funnel 8787
```

具体命令会随 Tailscale 客户端版本变化，请以当前 Tailscale 文档为准。

## 不推荐方案：路由器端口映射

不建议把 `8787` 直接映射到公网。

原因：

- 暴露面太大。
- 家用公网 IP 经常变化。
- 需要额外处理 HTTPS。
- 一旦口令泄露，风险更高。

## 我的建议

优先级：

1. 日常个人使用：Tailscale。
2. 必须公网域名访问：Cloudflare Tunnel + Cloudflare Access。
3. 临时演示：Cloudflare Quick Tunnel。
4. 不建议：路由器端口映射。
