# CallCodeX 使用说明

这是一个给 iPhone 使用的极简远程 Codex 客户端。它的工作方式是：Windows 电脑上运行 CallCodeX 网关服务，iPhone 通过 Tailscale 私有网络访问这个服务，然后把消息转发给 Windows 本地的 Codex。

## 1. 功能

- 在 iPhone 上通过 Safari 添加到主屏幕，像 App 一样使用。
- 选择 Windows 本地允许访问的项目目录。
- 输入文本并发送给 Windows 本地 Codex。
- 接收 Codex 的文本输出。
- 查看 Codex rate limits / 当前 goal token 信息。
- 不需要把服务暴露到公网，推荐只通过 Tailscale 使用。

## 3. Windows 端准备

### 3.1 安装依赖

进入项目目录：

```powershell
cd C:\Users\19719\Documents\CallCodeX
```

安装依赖：

```powershell
npm.cmd install
```

如果已经安装过，可以跳过。

### 3.2 配置访问口令

打开 `.env`：

```text
C:\Users\19719\Documents\CallCodeX\.env
```

访问口令由 `.env` 里的 `AUTH_TOKEN` 决定。不要把真实口令提交到 Git 或发给别人；需要在手机端手动输入时，直接从本机 `.env` 查看。

```text
AUTH_TOKEN=你的长随机字符串
```

长期使用前建议改成你自己的长随机字符串。

### 3.3 配置项目目录

打开 `projects.json`，把你希望在手机上选择的项目加入白名单：

```json
{
  "projects": [
    {
      "id": "callcodex",
      "name": "CallCodeX",
      "cwd": "C:\\Users\\19719\\Documents\\CallCodeX"
    }
  ]
}
```

字段说明：

- `id`：项目唯一标识，英文和数字即可。
- `name`：手机界面显示的项目名。
- `cwd`：Windows 本地项目目录。

手机端只能选择这里列出的目录，不能任意访问电脑上的其他路径。

## 4. 启动服务

### 4.1 启动 CallCodeX 网关

在项目目录运行：

```powershell
npm.cmd start
```

如果提示：

```text
Error: listen EADDRINUSE: address already in use 0.0.0.0:8787
```

说明 CallCodeX 网关已经在运行，不能重复启动同一个端口。你可以直接继续用手机访问。

如果你想重启服务，运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\restart-server.ps1
```

如果想后台启动，也可以运行：

```powershell
Start-Process -FilePath npm.cmd -ArgumentList 'start' -WorkingDirectory 'C:\Users\19719\Documents\CallCodeX' -WindowStyle Hidden
```

### 4.2 启动 Codex app-server

CallCodeX 需要连接本机 Codex app-server。目标命令是：

```powershell
codex.cmd app-server --listen ws://127.0.0.1:4500
```

如果 PowerShell 提示：

```text
codex : 无法将“codex”项识别为 cmdlet、函数、脚本文件或可运行程序的名称
```

说明 Codex CLI 没有安装，或者没有进入当前 PowerShell 的 `PATH`。Codex 桌面版 App 不一定等于可在终端里直接运行的 Codex CLI。

先运行检查脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check-codex-cli.ps1
```

如果它提示没有找到 Codex CLI，请按官方 Windows 安装方式运行：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"
```

安装完成后，关闭当前 PowerShell，重新打开一个新的 PowerShell，然后验证：

```powershell
codex.cmd --version
```

能输出版本号后，再启动 app-server：

```powershell
codex.cmd app-server --listen ws://127.0.0.1:4500
```

官方文档位置：

```text
https://developers.openai.com/codex/quickstart
https://developers.openai.com/codex/app-server
```

当前这台机器上，CallCodeX 网关已经能连接并初始化 Codex app-server。你仍然可以用下面的命令单独验证 Codex CLI：

```text
codex.cmd --version
```

如果你用其他方式启动了 Codex app-server，请在 `.env` 里设置：

```text
CODEX_APP_SERVER_URL=ws://127.0.0.1:4500
```

默认沙盒权限是：

```text
CODEX_SANDBOX=workspace-write
```

可选值通常包括：

```text
read-only
workspace-write
danger-full-access
```

## 5. 配置 Tailscale

### 5.1 Windows

1. 安装 Tailscale Windows 客户端。
2. 登录你的 Tailscale 账号。
3. 确认 Windows 设备在线。

### 5.2 iPhone

1. 从 App Store 安装 Tailscale。
2. 登录同一个 Tailscale 账号。
3. 打开 Tailscale VPN。

这样 iPhone 和 Windows 会处在同一个私有加密网络里，不需要公网端口映射。

## 6. 获取手机访问地址

在 Windows 项目目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\show-url.ps1
```

它会打印类似这样的地址：

```text
http://100.x.x.x:8787/?token=你的访问口令
```

在外面使用时，请优先选择 `100.x.x.x` 这种 Tailscale 地址。

## 7. iPhone 安装使用

1. 确认 iPhone 已连接 Tailscale。
2. 用 Safari 打开上一步打印出的地址。
3. 页面打开后，确认能看到 `CallCodeX`。
4. 点击 Safari 底部分享按钮。
5. 选择“添加到主屏幕”。
6. 以后从主屏幕打开 `CallCodeX` 即可。

第一次打开时，如果访问口令没有自动填入，就手动输入 `.env` 里的 `AUTH_TOKEN`，然后点“保存”。

## 8. 日常使用流程

1. Windows 电脑保持开机、联网。
2. Windows 上保持 Tailscale 在线。
3. Windows 上运行 CallCodeX 网关：

   ```powershell
   npm.cmd start
   ```

4. Windows 上运行 Codex app-server：

   ```powershell
   codex.cmd app-server --listen ws://127.0.0.1:4500
   ```

5. iPhone 打开主屏幕上的 CallCodeX。
6. 选择项目目录。
7. 点击“开启”。
8. 在输入框里输入给 Codex 的消息。
9. 点击“发送”。
10. 等待 Codex 输出显示在页面里。

## 9. 查看 tokens / 用量

点击右上角 `%` 按钮。

页面会尝试读取：

- Codex rate limits。
- 当前 thread goal 的 token 使用情况。

如果 Codex app-server 未连接，这里会显示错误信息。

## 10. 常见问题

### 页面打不开

检查：

- Windows 网关是否启动。
- iPhone 是否连接 Tailscale。
- 地址是否使用 Windows 的 Tailscale `100.x.x.x` IP。
- Windows 防火墙是否阻止了端口 `8787`。

### 显示 Codex 未连接

说明 CallCodeX 网关能打开，但还没连上 Codex app-server。

检查：

```powershell
codex.cmd app-server --listen ws://127.0.0.1:4500
```

是否能正常启动。

### PowerShell 不允许运行脚本

使用这个命令运行地址脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\show-url.ps1
```

### npm 命令报 `npm.ps1 cannot be loaded`

使用：

```powershell
npm.cmd install
npm.cmd start
```

不要直接用 `npm install` 或 `npm start`。

### 不想暴露到公网

不要做路由器端口映射。

推荐只用：

```text
iPhone -> Tailscale -> Windows -> CallCodeX -> Codex
```

## 11. 关闭服务

如果是在前台运行 `npm.cmd start`，按：

```text
Ctrl + C
```

如果是后台启动，可以在 PowerShell 里查找并结束：

```powershell
Get-CimInstance Win32_Process | Where-Object { $_.CommandLine -like '*server/index.mjs*' } | Select-Object ProcessId,CommandLine
Stop-Process -Id 进程ID
```
